Key Definitional Differences & Implications

🔴 AHT — Cisco Excludes Hold Time

Cisco AHT = Talk + ACW only. Genesys & Amazon include Hold.

  • Cisco will always show lower AHT than Genesys/Amazon
  • A center with 60s avg hold → Cisco AHT is ~60s lower
  • For apples-to-apples: compare "Inbound Contact Total Time" in Cisco to AHT in others
  • Agent Statistics report in Cisco does include Hold: (Hold + Connected + Wrapup) / Handled

🔴 Service Level — Denominator Differs

The single biggest reporting discrepancy.

  • Genesys: denominator = calls distributed (answered) → higher SL%
  • Amazon: denominator = all contacts queued (incl. abandoned) → lower SL%
  • Cisco: percentage of answered contacts / contacts that arrived minus short calls
  • Example: 90 answered in threshold, 10 abandoned → Genesys ≈ 90%, Amazon ≈ 81.8%

🔴 Occupancy — Three Different Denominators

  • Genesys: weighted by Forecasted IV (demand-weighted, planning focus)
  • Cisco: divide by total login/shift time (includes non-productive time)
  • Amazon: divide by available time (excludes non-productive time)
  • Amazon denominator is smallest → highest occupancy % for same contacts
  • Cisco includes non-productive in denominator → lowest occupancy %

🟡 ASA — Weighting Difference

  • Genesys: weighted avg per 15-min timestep by calls distributed
  • Cisco: Total Ringing Duration / Ringing Count (simple avg)
  • Amazon: simple avg of enqueue-to-answer per contact
  • Results diverge during intraday volume spikes; Genesys weights busy periods more

🟡 Abandonment Rate — Denominator & Method

  • Genesys: "Abandons Factor" — weighted per timestep, denominator = calls distributed
  • Amazon: simple (abandoned / queued) %
  • Genesys denominator is answered contacts; Cisco/Amazon use total offered
  • High-abandon scenarios show largest gap between platforms

🟡 Genesys 15-Min Timestep Weighting

All Genesys WFM metrics aggregate via weighted averages over 15-min intervals, weighted by interaction volume. This means:

  • Busy periods drive the daily average more than quiet periods
  • Results differ from Cisco/Amazon simple averages during intraday variation
  • More accurate for staffing/planning but harder to reconcile manually

🟡 AHT — Cisco Has Two Formulas

Cisco Analyzer defines AHT differently across report types:

  • Agent Details / Site report: (ConnectedDuration + WrapupDuration) / ConnectedCount — no hold
  • Agent Statistics report: (Hold + Connected + Wrapup) / Handled — hold included
  • Must know which report you're reading to interpret Cisco AHT correctly

🟢 Talk Time & ACW — Generally Consistent

All three platforms define talk time (connected duration) and ACW (wrapup) similarly. Low variance except:

  • Cisco explicitly tracks Inbound vs Outdial ACW separately
  • Amazon's ACW logic defaults to 0 (not null) when no ACW occurs
  • Genesys handles ACW within the AHT timestep calculation

🟢 Cisco Most Granular for Agent States

Cisco Analyzer tracks more agent-state sub-metrics than the other two:

  • Separate Reserved (Ringing), Hold, Connected, Wrapup for Inbound AND Outdial
  • CTQ (Consult-to-Queue) Request and Answer tracked separately
  • Wellness Breaks metric (AI burnout detection)
  • Social channel variants of all agent metrics

🟢 Amazon Connect Has the Most Metrics (~200+)

Amazon Connect tracks many categories not in Genesys WFM or Cisco Analyzer:

  • AI Agent / Prompt / Tool / Session metrics (40+ metrics)
  • Contact Lens conversational analytics (talk %, interruptions, non-talk)
  • Case management, bot metrics, outbound campaign metrics
  • Flow / IVR execution metrics
  • Test case execution metrics

🟢 Genesys WFM — Scheduling & Forecasting Native

Genesys WFM has native scheduling metrics not found in Cisco or Amazon core docs:

  • Schedule Adherence % formula (NC, UNC, ST components)
  • IV Forecast Accuracy (forecast vs actual, % difference)
  • AHT Forecast Accuracy
  • Coverage (FTE), Optimal Coverage, Budget calculations
  • Number of Agents Scheduled vs Actual
Bottom Line: The three biggest discrepancies when comparing reports across platforms are Service Level (denominator methodology), AHT (Cisco excludes hold in most reports), and Occupancy (three different denominators). Always confirm the formula before benchmarking cross-platform.
